How AoE2Meta
Turns Replays Into Insights

From raw leaderboard data to win-rate statistics — every step of the data analysis pipeline explained.

1
🏆 Leaderboard Sampling Input

We fetch the top 2,500 ranked players from the Microsoft AoE2 DE Leaderboard API every day, focusing on 1800+ ELO to ensure competitive, representative games — not casual matches.

2,500players sampled
1800+ELO threshold
RM 1v1game mode
2
🔍 Match Discovery Fetch

For each sampled player, we query the AoE2Companion API to retrieve their recent match IDs — filtering only the last 30 days to keep statistics current with the latest patch.

30 daysrolling window
~15Kmatch IDs / day
dedupalready processed IDs skipped
3
📥 Replay Download Download

Each match ID is resolved to a .aoe2record file via the aoe.ms/replay endpoint — Microsoft's own replay server. No auth required. The file is a ZIP containing the full match recording.

aoe.ms/replayofficial MS endpoint
ZIP.aoe2record inside
~300KBavg file size
4
⚙️ Replay Parsing (mgz) Parse

The replay is parsed with mgz — the most mature Python library for AoE2 replay files. We extract every in-game action: unit training, building construction, age-up research, technology timing, and EAPM.

mgzPython parser
~3–8sper replay
all agesfull game extracted
100+ action typesunit · building · tech
5
🧠 Opening Classifier AI Logic

A rule-based classifier inspects the first 12 minutes of each player's actions to label their opening strategy. Feudal timing, first military unit type, and transition patterns are the primary signals.

first 12 minanalysis window
8 openingsDrush FC · Scout Rush · M@A…
feudal timekey signal
6
💾 Raw Storage (JSONL) Storage

Parsed results are appended to date-partitioned JSONL files in data/raw-replays/YYYY-MM-DD.jsonl. Every record is flushed every 50 replays as a checkpoint. Files older than 30 days are pruned automatically.

JSONLappend-only
checkpoint/50crash-safe flushes
30-day TTLauto-pruned
7
🦆 Aggregation (DuckDB) SQL

DuckDB scans all JSONL files in seconds and runs a single SQL query to group by mode → map → ELO bracket → civ → opening, counting wins and games. The 30-day filter is applied at query time.

DuckDBin-process SQL engine
< 5sfull rebuild time
5 ELO bucketsall · 1000-1400 · 1800+ …
8
📤 JSON Export Output

DuckDB results are serialised into strategy-stats.json — a nested structure keyed by mode → map → ELO → civ → opening. The Next.js server reads this file at request time with zero latency.

strategy-stats.jsonnested 5-level dict
~2 MBfull export
0ms latencyfile read, no API call
